funções em JS

são tipos de objetos com a incrivél capacidade de serem invocadas

características

  • Podem ser literais

    function myFunc(){ // corpo da função }

  • Em alguns casos, ocorre o hoisting

    Hoisting - capacidade do JS de 'içar uma function declaration'

    exemplo Hoisting

  • são objetos de primeira classe

    funções em js são tratadas como qualquer outro objeto. Isso significa que:

    • Passar uma função por parâmetro para outras funções(callback)
    • Podem ser atribuídas em propriedades de objetos(métodos)
    • Podem ser retornadas como resultado de outra função
    • Podem ter suas próprias propriedades

  • No ES5 funções são a única maneira de gerar escopo
  • Podem ser IIFE(Immediately invoked Function Expression)

    Evitam poluir o escopo global, gerando um escopo isolado

    (function(){
    // corpo
    })()

    Não vai precisar de IIFE se:

    • Usar um Bundle (Webpack, Parcel, outros)
    • Usar ESModules(browsers modernos)
    • Programar em Node

  • Podem ter propriedades internas como arguments e name
